Chapter 4 Networking & The Internet Definitions

Network Classifications

 Network classifications refer to the categorization of networks based on their size, scope, and purpose.
MAN A metropolitan area network is a network that covers a geographic area larger than a LAN but smaller than a WAN.
WAN A wide area network is a network that covers a large geographic area, typically spanning multiple cities, states, or even countries.

Network Models

 Network models refer to the various models or frameworks that define how networks operate and communicate
Bus Network A bus network is a network topology in which all devices are connected to a single communication line called the bus.
Star Network A star network is a network topology in which all devices are connected to a central hub or switch.

Protocols

 Network models refer to the various models or frameworks that define how networks operate and communicate
CSNA/CD Ethernet Carrier Sense Multiple Access/Collision Detection (CSMA/CD) is a networking protocol used in Ethernet networks to avoid collisions when two devices attempt to transmit data at the same time.
CSMA/CA Wifi Carrier Sense Multiple Access/Collision Avoidance (CSMA/CA) is a networking protocol used in wireless networks to avoid collisions when two devices attempt to transmit data at the same time.

Combining Networks

 Combining networks refers to the process of connecting multiple networks together to create a larger, more complex network.
Repeater A repeater is a networking device that amplifies and retransmits signals to extend the range of a network.
Bridge A bridge is a networking device that connects two LANs or segments of a LAN to enable communication between them.
Switch A switch is a networking device that connects devices on a LAN and directs traffic between them based on their MAC addresses.
Router A router is a networking device that connects two or more networks and directs traffic
Gateway A networking device that connects two networks using different protocols, such as a LAN and a WAN.

Client/server model

 The client/server model is a distributed computing architecture where a client makes requests to a server, and the server responds with data. In this model, the client is typically a user or an application that needs to access data or services, while the server is a computer or a group of computers that provide the requested data or services.
P2p: peer-to-peer A decentralized communications model in which each party has the same capabilities and either party can initiate a communication session.
Distributed system A system whose components are located on different networked computers, which communicate and coordinate their actions by passing messages to one another.
Cloud computing The on-demand availability of computer system resources, especially data storage and computing power, without direct active management by the user.

Internet Addressing

 Internet addressing is the system of assigning unique numerical identifiers to every device connected to the internet. This system enables devices to communicate with each other over the internet by exchanging packets of data.
Tier 1 The highest level of Internet service providers, which have direct connections to the global Internet backbone.
Tier 2 Regional Internet service providers that connect to Tier 1 providers and smaller ISPs, and often provide services to a specific geographic area.
Tier 3 Local Internet service providers that connect to Tier 2 providers and provide Internet access to individual homes and businesses.

IP address

 An IP (Internet Protocol) address is a unique numerical identifier assigned to every device that is connected to a computer network that uses the Internet Protocol for communication. It is used to identify and locate network devices, such as computers, servers, routers, and other networked equipment, on a network.
ICANN The Internet Corporation for Assigned Names and Numbers (ICANN) is a non-profit organization that is responsible for managing the Internet's domain name system (DNS). Learn more
DNS The Domain Name System (DNS) is a hierarchical decentralized naming system for computers, services, or other resources connected to the Internet or a private network.
TLD A top-level domain (TLD) is the last segment of a domain name, which follows the final dot in an Internet address. Examples of TLDs include .com, .edu, and .org.
Country-code TLDs Country-code top-level domains (ccTLDs) are TLDs that are specifically reserved for use by countries or territories. Examples of ccTLDs include .us for the United States and .jp for Japan. Learn more

Internet protocols

 Internet protocols are sets of rules and standards that define how devices on the Internet communicate with each other. Examples of Internet protocols include TCP/IP, HTTP, and FTP.
FTP File Transfer Protocol (FTP) is a standard network protocol used to transfer files from one host to another over a TCP-based network, such as the Internet.
HTTP Hypertext Transfer Protocol is the protocol used to transfer data over the World Wide Web. It defines how messages are formatted and transmitted, and how web servers and browsers should respond to various commands.
SMTP Simple Mail Transfer Protocol is a protocol used to send and receive email over the internet. It is a client-server protocol, which means that a client (such as an email program) sends a request to a server (an SMTP server) to send an email, and the server responds with a confirmation or an error message. SMTP is one of the oldest internet protocols and is still widely used today.

World Wide Web

 The World Wide Web is a system of interlinked hypertext documents accessed through the Internet. It was invented by British computer scientist Tim Berners-Lee in 1989 while working at CERN in Switzerland.
URL A Uniform Resource Locator is a reference to a web resource that specifies its location on the Internet and the protocol for accessing it, such as HTTP or FTP. It consists of a protocol identifier, followed by a colon and two slashes, and then the resource's address.
Client-side activities Client-side activities refer to actions that take place on the user's device, such as rendering webpages, executing scripts, and managing user input.
Server-side activities Server-side activities refer to actions that take place on the web server, such as processing requests, accessing databases, and generating dynamic content.

Internet four-layer software

 The Internet four-layer software refers to a conceptual model that divides the internet into four layers: the application layer, transport layer, network layer, and link layer.
Application layer The application layer is the top layer in the four-layer software model and includes protocols for specific applications, such as HTTP, FTP, and SMTP.
Transport layer The transport layer is responsible for managing data transport between applications and includes protocols such as TCP and UDP.
Network layer The network layer is responsible for routing data between networks and includes protocols such as IP.
Link layer The link layer is responsible for transmitting data between nodes on a network and includes protocols such as Ethernet and Wi-Fi.

Internet is

 The internet is a global network of interconnected computers and devices that use standardized communication protocols to exchange data.
Packet-switching network A packet-switching network is a network that breaks data into packets and routes them through the network based on the destination address.
Fault-tolerant Fault-tolerant refers to the ability of a system to continue operating in the event of a hardware or software failure.
Redundant Redundant refers to the presence of multiple copies or instances of a system component to increase reliability and availability.

Cybersecurity

 Cybersecurity refers to the practice of protecting internet-connected systems, including hardware, software, and data, from attack, damage, or unauthorized access.
Malware Malware is a term used to describe malicious software, including viruses, worms, Trojan horses, and spyware, designed to harm or exploit a system or its users.
Virus A virus is a type of malware that is designed to replicate and spread itself to other computers.
Worm A worm is a type of malware that can replicate itself and spread across a network without human intervention.
Trojan horse A Trojan horse is a type of malware that appears to be a legitimate software application but contains malicious code that can harm a system or steal data.
Phishing Phishing is a type of social engineering attack in which an attacker sends a message or email that appears to be from a reputable source in order to trick the recipient into providing sensitive information or clicking on a malicious link.
DDoS attack A Distributed Denial of Service (DDoS) attack is a type of cyberattack in which a network of compromised devices is used to flood a website or server with traffic in order to overwhelm it and cause it to become unavailable.

Protection

 Protection refers to the measures taken to safeguard a system or network against potential threats, including security controls, risk management, and incident response.
Firewall A firewall is a network security device that monitors and filters incoming and outgoing network traffic based on predefined security rules.
Spam filter A spam filter is a software program that identifies and blocks unwanted or unsolicited emails, known as spam.
Proxy server A proxy server is a server that acts as an intermediary between a client device and the internet to improve security, performance, and privacy.
Antivirus software Antivirus software is a type of computer program designed to prevent, detect, and remove malware infections on individual devices or networks.
Cryptography Cryptography is the practice of securing communication from adversaries by transforming messages into unreadable formats that can only be deciphered by those who possess the appropriate key or knowledge.
Public key encryption Public key encryption is a type of encryption that uses a pair of keys - one public and one private - to encrypt and decrypt messages, making it possible for two parties to communicate securely without sharing a secret key.
