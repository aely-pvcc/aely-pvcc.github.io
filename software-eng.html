Software Engineering Terms

Term Definition
Software Engineering The application of a systematic, disciplined, quantifiable approach to the development, operation, and maintenance of software
Integrated Development Environment (IDE) A software application that provides comprehensive facilities to computer programmers for software development
Association for Computing Machinery (ACM) A professional organization that aims to advance computing as a science and profession
Purpose of ACM To promote the advancement of computing and information technology, and to provide a forum for the exchange of information and ideas related to the field
Benefits of ACM Membership Access to research, networking opportunities, discounts on publications and events, and career resources
Student ACM Membership A membership category for students that offers reduced rates and special resources for student members
Institute of Electrical and Electronics Engineers (IEEE) A professional association for electronic engineering and electrical engineering, and related disciplines
Purpose of IEEE To advance technology for the benefit of humanity, and to promote the engineering profession
Benefits of IEEE Membership Access to research, networking opportunities, discounts on publications and events, and career resources
Student IEEE Membership A membership category for students that offers reduced rates and special resources for student members
Software Life Cycle: Development phase The phase of software development that includes requirements analysis, design, implementation, and testing
Requirements analysis The process of gathering and defining the requirements of the system to be developed
Design The process of defining the architecture, components, interfaces, and other characteristics of the system to be developed
Implementation the process of writing the code and developing the software system based on the specifications and design produced during the earlier stages of software development
Testing The process of evaluating a system or its component(s) with the intent to find whether it satisfies the specified requirements or not.
Modular implementation The practice of dividing a software system into smaller parts (modules) that can be independently developed, tested, and maintained.
Unified Modeling Language (UML) A standardized modeling language used to visualize, specify, construct, and document the artifacts of a software system.
Use case A description of a set of actions that a system or component can perform to produce an observable result for an actor.
Use case diagram A diagram that shows the relationships among actors and use cases within a system.
Data relationships: One-to-one A relationship between two entities where each entity instance in one entity class corresponds to exactly one entity instance in the other entity class.
Data relationships: One-to-many A relationship between two entities where each entity instance in one entity class corresponds to one or more entity instances in the other entity class.
Data relationships: Many-to-many A relationship between two entities where each entity instance in one entity class corresponds to one or more entity instances in the other entity class, and vice versa.
Design pattern A reusable solution to a commonly occurring problem in software design.
Software quality assurance (SQA) The process of monitoring and improving the software engineering process to ensure quality.
Software testing: glass box testing A testing technique that examines the internal structure of a software system.
Software testing: black box testing A testing technique that examines the external behavior of a software system.
Software testing: beta testing A testing phase where a version of the software is released to a limited audience to collect feedback and identify bugs.
Software testing: alpha testing A testing phase where a version of the software is tested by the development team before release.
Documentation: user documentation Documentation that provides information and instructions to users on how to use a software product.
Documentation: system documentation Documentation that provides information about the technical aspects of a software system.
Documentation: technical documentation Documentation that provides information about the technical aspects of a software system.
Intellectual property law A set of laws that protect the rights of creators and owners of intellectual property, such as software.
Software license An agreement between the software owner and the user that specifies how the software can be used.
Software disclaimer A statement that limits the liability of the software owner for damages caused by the use of the software.
Best defenses against software liability claims Proper documentation, testing, and quality assurance processes can help defend against software liability claims.
