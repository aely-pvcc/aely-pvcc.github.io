Programming Languages

Term Definition
Machine Language The lowest-level programming language consisting of binary code that can be directly executed by a computer's CPU.
Debugging The process of identifying and fixing errors or bugs in a software program.
Assembler A program that translates assembly language code into machine language code.
Assembly Language A low-level programming language that uses mnemonic codes to represent machine language instructions.
Translator A program that translates code from one language to another.
Compiler A program that translates high-level programming language code into machine language code.
Interpreter A program that directly executes high-level programming language code without first compiling it.
Cross-Platform Software Software that can run on multiple operating systems or platforms.
Natural Language A language spoken by humans, such as English or French.
Formal Language A language with a defined set of rules and syntax, such as a programming language.
Machine Independence The ability of software to run on different types of computer hardware without modification.
High-Level Programming Language A programming language that is easier for humans to read and write than low-level programming languages.
Object-Oriented Programming (OOP) A programming paradigm that uses objects to represent and manipulate data and behavior.
Scripting Language A programming language used for creating scripts or small programs that automate tasks or enhance functionality.
Data Types: Integer A type of data that represents whole numbers without decimals, such as -3, -2, -1, 0, 1, 2, 3.
Float A numeric data type in programming that represents a real number with decimal point. Floats can be used to represent values such as pi or the square root of 2.
String A sequence of characters, often used to represent text in programming. Strings can be enclosed in single or double quotes.
Primitive Data Type A basic data type provided by a programming language, such as integer, floating point, or boolean. Primitive data types are not composed of other data types.
Data Structure A way of organizing and storing data in a programming language, such as arrays or linked lists. Data structures can be composed of primitive data types or other data structures.
Python Collection Data Structures Data structures provided by the Python programming language to organize and store data, such as lists, tuples, sets, and dictionaries.
List A collection data type in Python that is ordered and changeable. It allows duplicate elements.
Tuple A collection data type in Python that is ordered and unchangeable. It allows duplicate elements.
Set A collection data type in Python that is unordered and unindexed. It does not allow duplicate elements.
Dictionary A collection data type in Python that is unordered, changeable, and indexed. It does not allow duplicate elements.
Function A block of code in programming that performs a specific task and can be called multiple times throughout a program. Functions can have parameters and can return values.
Local Variable A variable in programming that is declared inside a function and can only be accessed within that function.
Global Variable A variable in programming that is declared outside a function and can be accessed throughout the program.
Source Program The original program code written by a programmer in a high-level language, before it is translated into machine code. The source program is compiled or interpreted to create the executable program.
Object Program A machine-executable code that is produced by compiling or assembling a source code program written in a high-level programming language.
