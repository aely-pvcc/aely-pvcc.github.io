Chapter 3 Operating Systems Terminology
Term Definition
Operating Sytem -The software that controlls the overall operation of a computer. It provides the means by which a user can store and retrieve files, provides the interface by which a user can request the execution of programs, and provides the environment necessary to execute the programs requested. Ex. One of the best known examples of a operating system is Windows.
Job Queue -The jobs residing in mass storage wait for execution in a Job Qeue. A queue is a storage organization in which objects(in this case, jobs) are ordered in first-, first-out (abbreviated FIFO and pronounced ("FI-foe") fasion. Most Job Queues dont rigorously follow the FIFO structure, since most operating systems provide for consideration of job priorities.
FIFO -First in first out.
Interactive Processing -New operating system that allowed a program to carry a dialogue with the user through remote terminals.
Time-sharing -Operating system that provided service to multiple users at the same time.
Multiprogramming -When time is divided into intervals and then the execution of each job is restricted to only one interval at a time.
Application Software -Programs that perform tasks particular to the machine's utilization.
Utility Software -Software units that extend the capabilities of the operating system.
Graphical User Interface (GUI) -Objects to be manipulated such as programs and files are represented with a picture display icon.
Shells -Older user interfaces that communicate with users through textual messages using a keyboard and monitor.
Kernel -The internal part of an operating system.
Device driver -Software units that communicate with the controllers to carry out operations on the peripheral devices attached to the machine.
File manager -A unit that's job is to coordinate the use of the machines mass storage facilities.
Memory manager -A unit in charge of coordinating the machine's use of main memory.
Virtual memory -A large “fictional” memory space created by paging.
Bootstrapping -Often shortened to booting is performed by a computer each time it is turned on. It transfers the operating system from mass storage to the main memory.
ROM -Stands for read-only memory; its contents can be read but can’t be altered.
Deadlock -The condition in which two or more processes are blocked from progressing because each is waiting for a resource that's allocated from another.
Spooling -The technique of holding data for output at a later and more convenient time.
Super user (or administrator) -Accounts are established by them. This person has highly privileged access to the operating system by identifying himself usually by a username and password. Once logged in, the administrator can change the settings within the operating system, adjust privileges given to other users, modify software packages, and perform various maintenance activities that other users cannot.
Multi-factor authentication -A solution to the weak passwords people create to secure their computer accounts that requires more than one piece of evidence to gain access.
